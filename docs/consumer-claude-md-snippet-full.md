## UI Components (@tale-ui/react)

This project uses `@tale-ui/react` for all UI components.

Before generating or modifying component code, you MUST:

0. **Plan before generating JSX.** If the Tale UI MCP server is available, call the `plan_ui` tool with the UI description first. It returns which components to use, a matching recipe if one exists, and key pitfalls — so you choose the right components before writing a single line of JSX. Skip this step only if you are making a trivial single-component change.

1. **Read the setup guide** in `node_modules/@tale-ui/react/README.md` — it contains critical configuration (font-size base, style imports, dark mode, theme overrides) that will produce broken output if skipped.

2. **Check the JSDoc `@example` on each component's .d.ts export** before using it. Every component's Root export includes a `@example` block showing the correct import path, sub-parts, and composition pattern. Read the `.d.ts` file for each component you intend to use:

   ```text
   node_modules/@tale-ui/react/esm/{name}/{Name}.styled.d.ts
   ```

   > **Exception:** A few components use `{Name}.d.ts` (not `.styled.d.ts`): `CSPProvider`, `CheckboxGroup`, `RadioGroup`, `Container`, and `mergeProps`.

   The `@example` block at the top of each file is the authoritative usage reference.

3. **For deeper details** (all props, all variants, advanced patterns), read the local component doc:

   ```text
   node_modules/@tale-ui/react/docs/{name}.md
   ```

4. **Do not guess component APIs.** Always check the `@example` block first. Incorrect usage (wrong sub-part names, missing wrapper components, wrong import paths) causes build failures.

There is NO `EmptyState.Header` or `EmptyState.Content` — place title and description directly inside `EmptyState.Root`. `EmptyState.Icon` can contain an `<Icon>` component or be omitted. `EmptyState.Actions` wraps action buttons below the description. Example: `<EmptyState.Root><EmptyState.Icon><Icon icon={Search} size="lg" /></EmptyState.Icon><EmptyState.Title>No results</EmptyState.Title><EmptyState.Description>Try adjusting your search terms or filters</EmptyState.Description><EmptyState.Actions><Button variant="primary">Reset filters</Button></EmptyState.Actions></EmptyState.Root>`. Using `<EmptyState>` directly will cause TypeScript errors.

The `Fieldset.Legend` sub-part renders the `<legend>` element that labels the group. If you need descriptive text below the legend, place a plain `<Text>` element (from `@tale-ui/react/text`) as a direct child of `Fieldset.Root`.    - **`Field.Root` does NOT accept validation state props** — `Field.Root` is a plain `div` wrapper (`HTMLAttributes<HTMLDivElement>`) and does not accept `isInvalid`, `isRequired`, `isDisabled`, or `isReadOnly`. Passing these to `Field.Root` causes 'is not assignable to type' TypeScript errors. Place validation state props on the inner control instead (e.g., `TextField.Root isInvalid`). `Field.Error` becomes visible when the inner control reports an invalid state. Correct structure: `<Field.Root><Field.Label>Email</Field.Label><Field.Description>We'll never share your email.</Field.Description><TextField.Root isRequired isInvalid><TextField.Input placeholder="you@example.com" /></TextField.Root><Field.Error>Please enter a valid email address.</Field.Error></Field.Root>`. **When composing `Field.Root` with `TextField.Root` inside it, both must be imported separately** — `Field` from `@tale-ui/react/field` and `TextField` from `@tale-ui/react/text-field`. Missing either import causes 'Cannot find name' TypeScript errors. For a simple field without Tale UI's validation features, use a plain `<input className="tale-input">` inside `<Field.Control>` — this requires only the `Field` import. **There are NO standalone `FieldLabel`, `FieldError`, `FieldControl`, or `FieldDescription` components** — these are sub-parts of the `Field` namespace only (`Field.Label`, `Field.Error`, `Field.Control`, `Field.Description`). Importing `{ FieldLabel, FieldError } from '@tale-ui/react'` or any other path causes "has no exported member 'FieldLabel'" TypeScript errors.
   - **`FileTrigger` is a simple component** — use `<FileTrigger>` directly with `onSelect` prop to handle file selection. Pass a `<Button>` or other trigger element as children. There is NO `FileTrigger.Root` — using it will cause "Property 'Root' does not exist" TypeScript errors. Example: `<FileTrigger onSelect={(fileList) => { if (fileList && fileList.length > 0) { console.log(fileList[0].name); } }}><Button>Choose File</Button></FileTrigger>`. The `onSelect` callback receives a `FileList | null` — **always null-check before accessing `fileList.length` or `fileList[0]`**; accessing either without a null guard causes 'fileList is possibly null' TypeScript errors.
   - **`GridList` namespace structure** — `GridList.Root` contains `GridList.Item` children for selectable grid layouts. There is NO standalone `GridList` component — using `<GridList>` directly will cause "Property 'Root' does not exist" TypeScript errors. Use `GridList.Root` with `selectionMode="multiple"` for multi-select functionality. **`GridList.Item` does NOT accept an `isSelected` prop** — passing `isSelected` causes 'is not assignable to type GridListItemProps' TypeScript errors. Selection state is managed entirely by `GridList.Root` via `selectedKeys` (a `Set` of selected ids) and `onSelectionChange` — do NOT maintain a separate `useState` array and pass `isSelected` to each item. **Do NOT add a separate `Checkbox` component inside `GridList.Item` to indicate selection** — the grid list handles selection styling automatically; adding a `Checkbox` is redundant and broken (it would require its own state and `Checkbox` is a namespace component requiring `Checkbox.Root`). For controlled selection, use `selectedKeys` and `onSelectionChange` on `GridList.Root`: `const [selected, setSelected] = useState<Set<string>>(new Set()); ... <GridList.Root selectionMode="multiple" selectedKeys={selected} onSelectionChange={(keys) => setSelected(keys as Set<string>)}>`. **Do NOT annotate the `onSelectionChange` callback parameter as `Set<string>`** — the actual type is `Selection` (a React Aria union type), not `Set<string>`. Writing `onSelectionChange={(keys: Set<string>) => ...}` causes "Type '(keys: Set<string>) => void' is not assignable to type '(keys: Selection) => void'" TypeScript errors. Always use an untyped arrow function and cast inside: `(keys) => setSelected(keys as Set<string>)`. For uncontrolled multi-select, simply use `selectionMode="multiple"` with no additional state. Example: `<GridList.Root selectionMode="multiple"><GridList.Item id="1">Item 1</GridList.Item><GridList.Item id="2">Item 2</GridList.Item></GridList.Root>`.
   - **`Menu` requires `Menu.MenuList` between `Menu.Popover` and `Menu.Item`** — `Menu.MenuList` renders the menu container that holds menu items. Placing `Menu.Item` directly inside `Menu.Popover` (without `Menu.MenuList`) will cause "Property 'List' does not exist" TypeScript errors. The correct structure is: `<Menu.Root><Menu.Trigger className="tale-button tale-button--neutral tale-button--md">File</Menu.Trigger><Menu.Popover><Menu.MenuList><Menu.Item id="new" onAction={() => console.log('New')}>New</Menu.Item></Menu.MenuList></Menu.Popover></Menu.Root>`. To group items with a separator, place `Menu.Separator` **inside** `Menu.MenuList` between items — use a single `Menu.MenuList` for all items and separators. Do NOT omit `Menu.MenuList` — it is required for proper menu rendering and layout. **`Menu.Separator` goes INSIDE `Menu.MenuList`, not as a sibling of it** — placing `Menu.Separator` between two `Menu.MenuList` elements is wrong. **A complete menu with a trigger button follows this exact pattern:** `<Menu.Root><Menu.Trigger className="tale-button tale-button--neutral tale-button--md">File</Menu.Trigger><Menu.Popover><Menu.MenuList><Menu.Item id="new" onAction={() => console.log('New')}>New</Menu.Item><Menu.Item id="open" onAction={() => console.log('Open')}>Open</Menu.Item><Menu.Separator /><Menu.Item id="save" onAction={() => console.log('Save')}>Save</Menu.Item><Menu.Separator /><Menu.Item id="close" onAction={() => console.log('Close')}>Close</Menu.Item></Menu.MenuList></Menu.Popover></Menu.Root>` — note `Menu.Trigger` uses `className` (not `asChild`) and all items share one `Menu.MenuList`.
   - **`Text` component basic usage for muted secondary text** — to create muted secondary text, use `<Text color="muted">This action cannot be undone.</Text>`. The `color="muted"` prop renders text in `--neutral-60` for secondary information. Always import `Text` explicitly: `import { Text } from '@tale-ui/react/text';`. Example: `<Text color="muted">This action cannot be undone.</Text>`.
   - **`Pagination` namespace structure** — `Pagination` is a compound namespace component that must be composed manually. Sub-parts: `Root`, `Item`, `Ellipsis`, `PreviousTrigger`, `NextTrigger`, `Dot`, `Line`. **`Pagination.Root` does NOT accept `currentPage`, `totalPages`, or `onPageChange` props** — passing these causes 'is not assignable to type RootProps' TypeScript errors. Instead, render each page button as `<Pagination.Item page={N} />` and mark the active page with the `current` prop: `<Pagination.Item page={3} current />`. Use `<Pagination.Ellipsis />` for truncated ranges and `<Pagination.PreviousTrigger />` / `<Pagination.NextTrigger />` for prev/next buttons (self-closing). Example showing page 3 of 10: `<Pagination.Root aria-label="Pagination"><Pagination.PreviousTrigger /><Pagination.Item page={1} /><Pagination.Item page={2} /><Pagination.Item page={3} current /><Pagination.Ellipsis /><Pagination.Item page={10} /><Pagination.NextTrigger /></Pagination.Root>`. For dot indicators use `<Pagination.Dot page={N} current />` and for step indicators use `<Pagination.Line page={N} current />`.
   - **`PinInput` namespace structure** — `PinInput.Root` contains `PinInput.Group` with `PinInput.Slot` children for each digit. There is NO `PinInput.Input` sub-part — using `<PinInput.Input />` will cause "Property 'Input' does not exist" TypeScript errors; the correct sub-part is `PinInput.Slot`. **`PinInput.Root` requires a `maxLength` prop** (total number of digit slots) — omitting it causes "Property 'maxLength' is missing" TypeScript errors. Each `PinInput.Slot` requires an `index` prop (zero-based). **`PinInput` has its own `PinInput.Separator` sub-part** for the visual divider between groups — do NOT use the standalone `Separator` component from `@tale-ui/react/separator`. Example for a 6-digit PIN split into two groups of 3: `<PinInput.Root maxLength={6}><PinInput.Group><PinInput.Slot index={0} /><PinInput.Slot index={1} /><PinInput.Slot index={2} /></PinInput.Group><PinInput.Separator /><PinInput.Group><PinInput.Slot index={3} /><PinInput.Slot index={4} /><PinInput.Slot index={5} /></PinInput.Group></PinInput.Root>`.

For a primary button that says "Save", use: `<Button variant="primary">Save</Button>`. The tab button sub-part is `Tabs.Tab` (with an `id` prop), and the content area sub-part is `Tabs.Panel` (with a matching `id` prop). **`Tabs.Root` uses `defaultSelectedKey` (NOT `defaultValue`) for the initially selected tab, and `selectedKey`/`onSelectionChange` for controlled state** — passing `defaultValue` causes "is not assignable to type RootProps" TypeScript errors. Example: `<Tabs.Root defaultSelectedKey="overview"><Tabs.List><Tabs.Tab id="overview">Overview</Tabs.Tab><Tabs.Tab id="features">Features</Tabs.Tab><Tabs.Tab id="pricing">Pricing</Tabs.Tab><Tabs.Indicator /></Tabs.List><Tabs.Panel id="overview"><Text>Overview content.</Text></Tabs.Panel><Tabs.Panel id="features"><Text>Features content.</Text></Tabs.Panel><Tabs.Panel id="pricing"><Text>Pricing content.</Text></Tabs.Panel></Tabs.Root>`. `Tabs.Indicator` is an optional animated bar placed inside `Tabs.List` after the tab buttons.
   - **Use `<Image>` instead of `<img>`, and `<Row>` instead of a flex `<div>` for horizontal layouts** — `Image` (from `@tale-ui/react/image`) is a simple component and the correct replacement for native `<img>` tags in Tale UI code: `<Image src="https://i.pravatar.cc/150" alt="Sarah Chen" radius="full" />`. Do NOT write `<img src="..." style={{ borderRadius: '50%' }} />` — use `<Image>` with its `radius` prop instead. **`Image` uses `radius` (NOT `borderRadius`) for rounded corners** — passing `borderRadius` causes 'is not assignable to type ImageProps' TypeScript errors because that prop does not exist. The `radius` prop accepts `'none'`, `'sm'`, `'md'`, `'lg'`, or `'full'` — NOT the standard Tale UI token suffixes (`'s'`, `'m'`, `'l'`). Use `radius="md"` for medium rounded corners and `radius="full"` for circular images. **`Image` does NOT accept a `size` prop** — passing `size={40}` (or any numeric/string size value) causes 'is not assignable to type ImageProps' TypeScript errors because that prop does not exist on `Image`. Use `width` and `height` props instead to control dimensions: `<Image src="/photo.jpg" alt="A mountain landscape" radius="md" width={40} height={40} />`. `Row` (from `@tale-ui/react/row`) is the correct replacement for `<div style={{ display: 'flex', gap: '...' }}>` when arranging children horizontally: `<Row gap="s"><Badge variant="brand">React</Badge><Badge variant="brand">TypeScript</Badge></Row>`. Do NOT use a bare `<div style={{ display: 'flex' }}>` where `<Row>` would serve — this is a primary cause of missing `Row` and `Image` components in generated output. Whenever you would write `display: 'flex'` on a `<div>`, use `<Row>` for horizontal layout or `<Column>` for a vertical/constrained block, with their respective imports.
   - **`Menu` requires `Menu.MenuList` between `Menu.Popover` and `Menu.Item`** — `Menu.MenuList` renders the menu container that holds menu items. Placing `Menu.Item` directly inside `Menu.Popover` (without `Menu.MenuList`) will cause "Property 'List' does not exist" TypeScript errors. 7. **Dark mode must persist between refreshes.** Every new app must:

   a. Add `class="tale-ui"` and `data-color-mode` to `<html>`:
   ```html
   <html class="tale-ui" data-color-mode="light">
   ```

   b. Include this inline script in `<head>` before any CSS or JS to avoid a flash of wrong theme:
   ```html
   <script>
     (function() {
       var mode = localStorage.getItem('color-mode')
         || (matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
       document.documentElement.setAttribute('data-color-mode', mode);
     })();
   </script>
   ```

   c. Use `ColorModeToggle` for the toggle UI — it handles `localStorage` and `data-color-mode` automatically:
   ```tsx
   import { ColorModeToggle } from '@tale-ui/react/color-mode-toggle';

   <ColorModeToggle />
   ```

9. **Theming with category tokens.** Override component families in one place by setting category tokens on `:root`. These tokens are defined in `@tale-ui/react-styles` and default to the same semantic token values they replace — so changing them shifts the entire family simultaneously:

   ```css
   :root {
     /* Retheme all form field inputs at once */
     --field-bg: var(--neutral-10);
     --field-border-color: var(--neutral-20);
     --field-radius: var(--radius-s);

     /* Retheme all dropdown popups and menus */
     --popup-bg: var(--neutral-12);
     --popup-radius: var(--radius-m);
     --popup-shadow: var(--shadow-l);

     /* Retheme all modal dialogs and drawers */
     --modal-title-color: var(--color-80);
     --modal-backdrop-bg: var(--scrim-strong);

     /* Retheme all progress bars and meters */
     --progress-track-height: 0.4rem;
     --progress-indicator-bg: var(--color-60);
   }
   ```
